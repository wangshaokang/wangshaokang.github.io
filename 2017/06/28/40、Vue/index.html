###MVC
1、m: model 数据层 保存数据
2、v: view 视图层 用户界面
3、c: controller 控制器 业务逻辑
过程：v-c-m-v
用户操作界面 通过DOM事件—->向控制器发送指令 执行相应 业务逻辑 —> 通知数据层 保存或者修改状态 再有数据层—->反馈给视图层

##angular 采用MVC模式 单向通信

##MVVM 双向数据绑定

  • m model 数据模型 保存数据
  • v view 视图层 用户界面
  • vm 视图模型 viewModel enter code here
    v <->vm <->m
    1、view 改变了 —->会导致VM改变 —-> 通知m数据源状态改变
    2、m 数据源跟新了 —> VM发生改变—-> v视图跟着改变

##Vue 压缩和不压缩
通常开发阶段 引入不压缩版 报错提示 便于调试 生成版（压缩版） Vue.min.js 是没有 报错提示的
