js中this当前行为的主体

1、全局中的this是window
主要研究函数中的this
2、函数执行的时候方法中的this要看函数执行的时候要看前面有没有”.”

  • 如果前面有点 点前面是谁 this就是谁
  • 如果没有点 this就是window
    3、函数中的this和函数在哪里执行没有关系 只看执行的时候
    4、自执行函数里面的this永远是window
    5、给元素绑定事件，事件触发的时候，里面的this 就是当前绑定事件的元素
    6、给定时器的this也是window

##单例模式 (缺点：类似手工作坊)
把描述同一个事物（同一个对象）的属性和方法放在一个内存空间 起到分组的作用 即使属性名相同也不会发生冲突

##工厂模式
把实现一个功能的代码 放到一个函数里面 减少冗长的代码 “函数封装” “高内聚低耦合” 减少页面中的重复代码 提高代码重复利用率

###模块化开发 ：
项目中 开发模块比较多的时候，多人协作 每个人负责一个模块 最终将代码合在一起 单例模式中把flightFn和trainFn成为“命名空间”

##面向对象
在很多语言中都有面向对象
对象是一个泛指

  • 类：对对象的具体细分
  • 实例：类中某一个具体事物
  • 类的封装 继承 多态
  • 继承 子类继承父类
    • 多态：多种状态 （重载和重写）

      JS中没有重载

      重写：子类重写父类

##构造函数
创建一个自定义类 并且创建这个类的实例
构造函数执行 它的返回值是一个实例

###构造函数模式执行的时候，如果指定返回的 是一个引用数据类型，那么返回的就是这个引用数据类型

###在类中的this.xxx是给 当前实例添加属性 类中的this是当前实例

###类本身是个函数 类都是函数数据类型

###实例就是对象数据类型

工厂模式和构造函数模式的区别
1、工厂模式 普通函数执行 构造函数通过new来执行 默认创建一个对象 将其返回的值作为这个类的实例
2、执行的时候形成一个私有作用域—》形参赋值—》预解释—》代码 从上到下执行
不同
构造函数模式执行的时候，会默认创建一个对象 默认返回的实例
工厂模式 自己手工创建

instanceof 用来检测一个实例是否属于这个类

###数字
parseFloat(.3)===> 0.3
parseFloat(.3px)===> NaN
parseInt(.3)===> 0
parseInt(.3px)===>NaN

##字符串（所见即所得）
parseFloat(‘.4px’)===>0.4
parseFloat(‘.px’)===>NaN
parseInt(‘.4’)===>NaN（所见即所得）
parseInt(‘.4px’)===>NaN（所见即所得）
